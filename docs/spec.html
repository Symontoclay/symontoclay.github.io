Introduction

back to top

This is manual for SymOntoClay DSL.

This document contains all implemented grammar. You can use It.

SymOntoClay DSL is in development now. So new parts will be added here right after implementation and internal testing. I will try to do this as soon as possible.

Notation

back to top

The syntax is specified using Extended Backus-Naur Form (EBNF).

Source code representation

back to top

There are two views of source code file: as set of characters and as set of code entities.

Symbols

back to top

Characters

back to top

Source code is Unicode text encoded in UTF-8.

Newline = /* the Unicode code point U+000A */ .
UnicodeChar = /* an arbitrary Unicode code point except Newline */ .

Letters and digits

back to top

The underscore character _ (U+005F) is considered a letter.

Letter = "A" ... "Z" | "a" ... "z" | "_" .
DecimalDigit = "0" … "9" .

Kind of source code files

back to top

There are different kinds of source code files which can contain different code entities.

SourceCodeFileContent = WorldSourceCodeFileContent | NPCSourceCodeFileContent | HostSourceCodeFileContent | ClassSourceCodeFileContent .

World source code file

back to top

The file contains World declaration. There can be only one World declaration in the file. Also this file can contain other content which is allowed in Class source code file.

WorldSourceCodeFileContent = [ ClassSourceCodeFileContent ] [ WorldDecl ] [ ClassSourceCodeFileContent ] .

NPC source code file

back to top

The file contains NPC declaration. There can be only one NPC declaration in the file. Also this file can contain other content which is allowed in Class source code file.

NPCSourceCodeFileContent = [ ClassSourceCodeFileContent ] [ NPCDecl ] [ ClassSourceCodeFileContent ] .

Host source code file

back to top

The file contains Host declaration. There can be only one Host declaration in the file. Also this file can contain other content which is allowed in Class source code file.

HostSourceCodeFileContent = [ ClassSourceCodeFileContent ] [ HostDecl ] [ ClassSourceCodeFileContent ] .

Class source code file

back to top

The file contains all possible code entities, except World, NPC and Host declarations. There exist special source code files for these code entities. But content of Class source code file can be included into World, NPC and Host source code files.

ClassSourceCodeFileContent = [ ClassDeclSet ] [ FactDeclSet ] [ LogicRuleDeclSet ] .

Lexical elements

back to top

Comments

back to top

SymOntoClay supports both single and multiline comments. It is a programmer-readable explanation or annotation of source code like in many other programming languages. Also It helps to cut temporarily unused code fragments.

SingleLineComment = "//" [{ UnicodeChar }] NewLine .
//@r = @b = 1;
MultiLineComment = "/*" [{ UnicodeChar }] "*/" .
/*@r = @b = 1;
@bx >> @>log;*/

Tokens

back to top

Tokens form the vocabulary of the SymOntoClay DSL. There are four classes: identifiers, keywords, operators and punctuation, and literals.

Identifiers

back to top

Identifiers name code entities.

Since SymOntoClay has many kinds of code entities, I have decided It will be better to recognize kind of code entity by identifier. So SymOntoClay has many kinds of identifiers with different prefixes. Each of these identifiers has Its own meaning in code.

Identifier = ConceptIdentifier | InstanceIdentifier | VarIdentifier | SysVarIdentifier | QuestionVarIdentifier | LogicRuleVarIdentifier | ChannelIdentifier .

Concept identifier

back to top

Names a concept.

ConceptIdentifier = (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
a
_a
a2
aB
ab_1
Great_Britain
`Great Britain`

Instance identifier

back to top

Names an instance of concept.

InstanceIdentifier = "#" (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
#123

Variable identifier

back to top

Names a variable.

VarIdentifier = "@" (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
@a

System variable identifier

back to top

Names a system variable

SysVarIdentifier = "@@" (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
@@host

Question variable identifier

back to top

Names a variable for describing select-pattern in select query. This variable represents information which should be found in the select query.

QuestionVarIdentifier = "?" (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
?x

Logic rule variable identifier

back to top

Names special variable for Logic rule.

LogicRuleVarIdentifier = "$" (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
$x

Channel identifier

back to top

Names Channel.

ChannelIdentifier = "@>" (( Letter { Letter | DecimalDigit } ) | ( "`" { UnicodeChar } "`")) .
@>log

Integer literal

back to top

An integer literal is a sequence of digits representing an integer constant. Now integer literal has only decimal base.

IntegerLiteral = DecimalDigit [ { DecimalDigit } ] .
42

Floating-point literal

back to top

A floating-point literal is a decimal or hexadecimal representation of a floating-point constant.

A floating-point literal consists of an integer part (decimal digits), a decimal point and a fractional part (decimal digits). Now floating-point literal has only decimal base.

FloatPointLiteral = ( DecimalDigit [ { DecimalDigit } ] ) "." ( DecimalDigit [ { DecimalDigit } ] ) .
36.6

String literal

back to top

A string literal represents a string constant obtained from concatenating a sequence of characters.

StringLiteral = (""" UnicodeChar [{ UnicodeChar }] """) | ("'" UnicodeChar [{ UnicodeChar }] "'") .
"abc"
'abc'

Objects declaration

back to top

Everything is an object. The classical type-instance relationship is using, but there is not strict and fundamental distinction between the two.

Some objects have a special meaning. So Class represents a general concept which can have many instances.

Some objects have special engine support to make writing code easier and faster, or as entry point locators.

Class

back to top

Represents a general concept which can have many instances.

In general, It is similar to the concept 'class' in other OOP languages.

ClassDeclSet = ClassDecl [{ ClassDecl }] .
ClassDecl = "class" ConceptIdentifier [ InheritanceDecl ]
"{"
[ ClassBodyContent [{ ClassBodyContent }] ]
"}" .
ClassBodyContent = [ FactDeclSet ] [ LogicRuleDeclSet ] [ TriggersContent ] .
class exampleClass
{
}
class exampleClass is human, [0.1] animal
{
}

NPC

back to top

It is root object and entry point of NPC.

NPCIdentifier = ConceptIdentifier | InstanceIdentifier .
NPCDecl = "npc" NPCIdentifier [ InheritanceDecl ]
"{"
[ NPCBodyContent [{ NPCBodyContent }] ]
"}" .
NPCBodyContent = [ [ FactDeclSet ] [ LogicRuleDeclSet ] [ TriggersContent ] ] .
npc PeaceKeeper
{
    on Init =>
    {
    }
}
npc PeaceKeeper is human
{
    on Init =>
    {
    }
}

World

back to top

It is root object of World.

WorldIdentifier = ConceptIdentifier | InstanceIdentifier .
WorldDecl = "world" WorldIdentifier [ InheritanceDecl ]
"{"
[ WorldBodyContent [{ WorldBodyContent }] ]
"}" .

world `Lost town`

}

back to top

"}" .

Waypoint

back to top

Represents a point in space, abstracting from the game engine's coordinate system.

Coordinates of points are presented in a more convenient way, similar to that is used by people in everyday life.

Now It is supported the coordinate system what is similar to Polar coordinate system. The radial coordinate is the distance from center of NPC to target. And the angular coordinate (in degrees only) starts from the sagittal plane of NPC in front of the NPC.

SymOntoClay engine provides mapping to the game engine's coordinate system.

WaypointDecl = "#@" "[" WaypointDistance "," WaypointAngleInDegrees "]" .
WaypointDistance = NumberExpr .
WaypointAngleInDegrees = NumberExpr .
#@[25, 30]

Fact

back to top

Represents knowledge in declarative way. It is an element of Logic programming.

Now the fact is a predicate record with small syntactic sugar for better inclusion into SymOntoClay DSL.

For example, expression "cat (#Alisa)" is equivalent to "#Alisa is a cat".

In the future, I plan to move away from the predicate form to a more human-readable one. In this case, the predicate form will be saved as an alternative way of describing.

FactDeclSet = FactDecl [{ FactDecl }] .
FactDecl = "{:" ( FactSection | FactSentence ) ":}" .
FactSection = [ ">:" ] "{" FactSentence "}" .
FactSentence = FactPredicate .
FactPredicate = ConceptIdentifier "(" FactPredicateParam [ { "," FactPredicateParam } ] ")" .
FactPredicateParam = ConceptIdentifier | InstanceIdentifier .
{: male(#Tom) :}
{: >:{ male(#Tom) } :}
{: parent(#Piter, #Tom) :}

Logic rule

back to top

LogicRuleDeclSet = LogicRuleDecl [{ LogicRuleDecl }] .
LogicRuleDecl = "{:" LogicRulePrimarySection ImplicationSymbol LogicRuleSecondarySection ":}" .
LogicRulePrimarySection = [ ">:" ] "{" LogicRuleSentence "}" .
LogicRuleSecondarySection = "{" LogicRuleSentence "}" .
ImplicationSymbol = "->" .
LogicRuleSentence = LogicRulePredicate [ { AndOpExpr LogicRulePredicate } ] .
LogicRulePredicate = ConceptIdentifier "(" LogicRulePredicateParam [ { "," LogicRulePredicateParam } ] ")" .
LogicRulePredicateParam = ConceptIdentifier | InstanceIdentifier | LogicRuleVarIdentifier .
{: {son($x, $y)} -> { male($x) & parent($y, $x) } :}
{: >: {son($x, $y)} -> { male($x) & parent($y, $x) } :}

Trigger

back to top

A trigger is a code that is automatically executed in response to certain events.

Now only Lifecycle triggers are available. But in the future I am going to develop other kinds of triggers.

TriggersContent = "on" LifecycleTriggerIdentifier "=>"
"{"
StatementsSet
"}" .

Life cycle triggers

back to top

Every object has a life cycle.

LifecycleTriggerIdentifier = InitLifecycleTriggerIdentifier .
Init

back to top

Initializes an object. It is like a constructor in other programming languages.

Init trigger of NPC is a first code which is executed for the NPC. So It is an entrypoint of the NPC.

InitLifecycleTriggerIdentifier = "Init" .

Expressions

back to top

An expression specifies the computation of a value by applying operators and methods to operands.

Expr = WaypointDecl | ConstExpr | VarExpr | SysVarExpr | ChannelExpr | AssignmentOpExpr | PointerOpExpr | IsOpExpr | UseIsOpExpr | LeftRightStreamOpExpr | SelectOpExpr | InsertOpExpr .

Constant expression

back to top

Constant expressions may contain only literals and are evaluated at compile time.

ConstExpr = ConceptIdentifier | InstanceIdentifier | VarIdentifier | SysVarIdentifier | ChannelIdentifier | StringLiteral | NumberExpr .
NumberExpr = IntegerLiteral | FloatPointLiteral .
25
30.2
"abc"

Variable

back to top

A variable is a storage location for holding a value.

Both reading and writing are allowed for variables.

Default value of variable is NULL.

VarExpr = VarIdentifier .
@x
@`target value`

System variable

back to top

It's a special variable, which value is only written by engine.

Only reading is allowed for system variables.

SysVarExpr = SysVarIdentifier .
@@host

Available system variables

back to top

@@self

back to top

Contains a link to the current NPC.

@@host

back to top

Contains a link to the C# methods of this NPC which are defined at the Unity3D level of the NPC logic.

Allows to call low-level C# methods for interaction with Unity3D.

It does nothing when It launched with the "CLI run".

Channel

back to top

Channels are a conduit through which you can send and receive values with the stream operator.

Now only sending has been implemented.

ChannelExpr = ChannelIdentifier .

Available system channels

back to top

@>log

back to top

A channel for writing information to the log in text form.

Logging targets are determined by the engine settings.
