Contributing

Help is always welcome and there are plenty of options how you can contribute to SymOntoClay.

In particular, your contribution would be helpful in the following areas:

  • Trying to use SymOntoClay.
  • Reporting issues.
  • Fixing and responding to SymOntoClay’s GitHub issues.
  • Improving the documentation, especially fixing grammar issues. I am not a native English speaker, so there are a lot of grammar mistakes in SymOntoClay's documentation.
  • Getting involved in the language design process by proposing language changes or new features in the SymOntoClay’s forum and providing feedback.
  • Getting involved in improving the design process: all ideas are welcome.
  • Getting involved in creating new features. There are also many sub-areas in the big area:
    • Implementing new language features
    • Implementing new engine features
    • Improving performance
    • Improving integration with Unity
    • Creating extensions for code editors. First of all SymOntoClay needs extensions for Visual Studio and Visual Studio Code.
    • Improving DevOps
    • Redesign the SymOntoClay’s site.

Please note that this project is released with a Contributor Code of Conduct. By participating in this project - in the issues, pull requests, or Discussion - you agree to abide by its terms.
